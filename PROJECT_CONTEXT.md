# 📁 ملف سياق مشروع: صندوق الاشتراكات الشهرية
> أنشئ هذا الملف لنقل المحادثة وحالة المشروع إلى Claude Code لمتابعة التطوير.

---

## 🎯 وصف المشروع

تطبيق هاتفي لإدارة **اشتراكات شهرية** بقيمة **10 دينار ليبي لكل فرد**.
يعمل بالكامل offline (`localStorage`)، مع طبقة اختيارية للنسخ الاحتياطي/الاستعادة السحابية عبر **OneDrive** عند توفر إنترنت. يُستضاف على رابط مجاني (GitHub Pages) ثم يُغلَّف كـ APK عبر WebIntoApp.com.

## 🏗️ البنية التقنية

```
index.html                  ← الملف الرئيسي (كل شيء داخله، مستضاف على GitHub Pages)
  ├── CSS مدمج              ← تصميم داكن، RTL، متجاوز + CSS طباعة
  ├── HTML                  ← 5 صفحات + نافذة modal + toast + شاشة قفل PIN
  └── JavaScript مدمج       ← منطق التطبيق الكامل + مصادقة/مزامنة OneDrive + مزامنة ويندوز
manifest.json                ← إعدادات PWA (اسم، أيقونة، standalone)
sw.js                        ← Service Worker: يخزّن index.html مؤقتاً لعمل offline موثوق
server.js                    ← خادم Node.js/Express للمزامنة المحلية (ويندوز)
main.js                      ← نقطة دخول Electron لتطبيق ويندوز
preload.js                   ← نصوص Electron المساعدة
package.json                 ← إعدادات المشروع والاعتماديات
AZURE_SETUP.md               ← دليل تسجيل تطبيق Azure AD + الاستضافة + بناء APK
subscription_offline.html    ← النسخة الأصلية بدون سحابة (محفوظة كمرجع تاريخي)
```

**ملاحظة معمارية مهمة:** التطبيق يبقى local-first تماماً — كل القراءة/الكتابة تمر عبر `localStorage` كما كان. طبقة OneDrive إضافة معزولة تماماً (نسخ احتياطي/استعادة يدوية أو تلقائية عند توفر إنترنت + مصادقة)، وليست مزامنة ثنائية الاتجاه معقدة — الجهاز الواحد هو مصدر الحقيقة، وOneDrive مجرد نسخة احتياطية.

### المتغيرات الرئيسية
```javascript
const STORE = "sub_v4";     // مفتاح localStorage لبيانات الصندوق (db)
const FEE = 10;             // الاشتراك الشهري للفرد (دينار)
const CONFIG = { MSFT_CLIENT_ID: "..." }; // Client ID من Azure (راجع AZURE_SETUP.md)
let db = {
  families: [],             // الأسر
  expenses: [],             // المصروفات
  deposits: [],             // الإيداعات
  carryOver: 0,             // رصيد مرحّل للصندوق
  settings: {
    orgName, startDate, adminPhone, adminEmail
  }
}
```

**مفاتيح localStorage إضافية (منفصلة عمداً عن `db` وعن التصدير/الاستيراد JSON):**
- `sub_pin` — رمز قفل التطبيق (إن فُعِّل).
- `msft_tokens` — access/refresh token الخاصة بـ Microsoft (لا تُصدَّر أبداً ضمن النسخة الاحتياطية JSON).
- `onedrive_meta` — وقت آخر نسخة احتياطية ناجحة إلى OneDrive.

### هيكل بيانات الأسرة
```javascript
{
  id, name, phone, phone2, email,
  whatsapp, messenger, telegram,
  bankName, accountNo, address, notes,
  startDate,        // "YYYY-MM" — بداية الاستحقاق
  carryOver,        // رصيد مرحّل من فترة سابقة
  credit,           // رصيد معلق (دفعة مقدمة / فائض)
  children: [{id, name, startDate}],  // startDate لكل ابن
  payments: {
    "2024-01": {date, amount},
    "2024-02": {date, amount},
    ...
  },
  createdAt
}
```

### الدوال الأساسية
```javascript
famFee(f)              // الاشتراك الشهري الحالي = FEE × (1 + عدد الأبناء)
famDueMonths(f)        // كل الشهور المستحقة من startDate حتى الآن
famActiveCount(f, m)   // عدد الأفراد النشيطين في شهر معين
famFeeForMonth(f, m)   // قيمة الاشتراك لشهر معين حسب الأعضاء النشيطين
famUnpaid(f)           // الشهور غير المدفوعة
famTotalDue(f)         // إجمالي المستحق (مراعياً تاريخ بداية كل ابن)
famBalance(f)          // الرصيد = مدفوع + مرحّل + رصيد معلق - المستحق
totals()               // إجماليات كل الأسر + مصروفات + إيداعات
monthsUntilYearEnd()   // الشهور المتبقية من الشهر الحالي حتى نهاية السنة (للدفع المقدّم)
```

### الصفحات (5 صفحات)
1. **home** — لوحة التحكم (إحصائيات + تنبيهات التأخر)
2. **families** — قائمة الأسر (بحث + تفاصيل + تعديل + دفع)
3. **finance** — المصروفات / الإيداعات / الملخص
4. **reports** — تقرير شهري مرئي + حالة الأسر
5. **settings** — إعدادات + نسخ احتياطي محلي + OneDrive + قفل PIN + استيراد

## 🔄 ملاحظة مهمة: آلية تحديث التطبيق (PWA Update)
- السيرفس ووركر يستخدم استراتيجية **network-first** للملاحة HTML - يحصل على آخر إصدار فور الاتصال بالإنترنت
- عند نشر تحديث جديد، يكتشف المتصفح التغيير في ملف `sw.js` ويقوم بتثبيته
- يظهر إشعار "🆕 نسخة جديدة متاحة!" مع زر "تحديث الآن"
- فحص دوري كل ساعة للتحقق من وجود تحديثات جديدة
- خيار `updateViaCache: 'none'` يمنع تخزين الـ SW نفسه مؤقتاً

## 🔑 ملاحظات مهمة للمطور
1. **لا تضف أي CDN أو مكتبة خارجية** — التطبيق يجب أن يبقى قابلاً للعمل offline بالكامل
2. **الخط**: `system-ui` المدمج في النظام (لا Google Fonts)
3. **البيانات**: كلها في `localStorage` — `sub_v4` لبيانات الصندوق، ومفاتيح منفصلة (`sub_pin`, `msft_tokens`, `onedrive_meta`) لا تُصدَّر أبداً ضمن ملف JSON الاحتياطي
4. **الاتجاه**: RTL كامل (`dir="rtl"`)
5. **الجهاز المستهدف**: هواتف أندرويد (max-width: 480px)
6. **الاستضافة الآن مطلوبة قبل التغليف**: `index.html` يُرفع إلى GitHub Pages (وليس رفعاً مباشراً لـ WebIntoApp) لأن مزامنة OneDrive تحتاج نداءات شبكة حقيقية لا تعمل بشكل موثوق من WebView تحمّل ملف `file://` محلي — التفاصيل الكاملة في `AZURE_SETUP.md`
7. **مزامنة OneDrive تستخدم Device Code Flow** (لا Redirect URI) لأنها تعمل داخل أي WebView مقيدة — يجب تعبئة `CONFIG.MSFT_CLIENT_ID` في `index.html` بعد تسجيل تطبيق Azure (راجع `AZURE_SETUP.md`)
8. **الأمان**: `Files.ReadWrite.AppFolder` هي أضيق صلاحية ممكنة — الوصول يقتصر على مجلد التطبيق الخاص داخل OneDrive، وليس كامل حساب المستخدم

## 🪟 تشغيل نسخة ويندوز (Electron + خادم محلي)

### المتطلبات
- [Node.js](https://nodejs.org) (الإصدار 18 أو أحدث) مثبّت على جهاز ويندوز

### طريقة التشغيل
```bash
# 1. تثبيت الاعتماديات (في مجلد المشروع)
npm install

# 2. تشغيل تطبيق ويندوز (الخادم + الواجهة)
npm start

# أو تشغيل الخادم فقط (للاتصال من الجوال)
npm run server
```

### طريقة التوزيع (بناء مثبّت Windows .exe)
```bash
npm run dist
# ملف التثبيت سيكون في مجلد dist/
```

### الاتصال من الجوال
1. شغّل الخادم على ويندوز (`npm run server`)
2. ستظهر رسالة بعنوان الخادم (مثال: `http://192.168.1.100:3456`)
3. في إعدادات التطبيق على الجوال → المزامنة مع ويندوز
4. أدخل العنوان وانقر "اختبار الاتصال"
5. استخدم "رفع إلى الخادم" للمزامنة
